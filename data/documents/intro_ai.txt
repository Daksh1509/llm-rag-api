Artificial Intelligence (AI) is the simulation of human intelligence in machines
that are programmed to think and learn. AI systems can perform tasks such as
visual perception, speech recognition, decision-making, and language translation.
AI is broadly categorized into Narrow AI (designed for specific tasks) and
General AI (human-level reasoning across all domains).

Machine Learning (ML) is a subset of AI that allows computers to learn from data
without being explicitly programmed. Instead of writing specific rules, ML systems
learn patterns from examples and improve over time. Common ML algorithms include
linear regression, decision trees, random forests, support vector machines,
and neural networks.

Deep Learning is a type of machine learning that uses neural networks with many
layers (hence "deep"). It excels at tasks like image recognition, natural language
processing, and speech recognition. Popular deep learning frameworks include
PyTorch and TensorFlow.

Neural Networks are computing systems inspired by the human brain. They consist
of layers of interconnected nodes (neurons) that process information. The input
layer receives data, hidden layers transform it, and the output layer produces
predictions. Training involves adjusting weights using backpropagation and
gradient descent to minimize a loss function.

Natural Language Processing (NLP) is the ability of computers to understand,
interpret and generate human language. Applications include chatbots, machine
translation, sentiment analysis, named entity recognition, and text summarization.
Transformer models like BERT and GPT have revolutionized NLP since 2017.

Large Language Models (LLMs) are deep learning models trained on massive text
datasets to understand and generate human language. Examples include OpenAI GPT-4,
Google Gemini, Meta LLaMA, and Mistral. LLMs use the Transformer architecture
with self-attention mechanisms to process and generate text token by token.

Retrieval-Augmented Generation (RAG) is an AI technique that combines document
retrieval with language model generation. When a user asks a question, the system
first retrieves relevant document chunks using vector similarity search, then
passes those chunks as context to the LLM to generate a grounded answer.
RAG prevents hallucination and grounds the model in real, document-specific facts.
RAG is composed of two phases: indexing (offline) and retrieval plus generation (online).

The RAG pipeline works as follows:
Step 1 - Load raw documents (PDF, TXT, DOCX formats).
Step 2 - Split documents into smaller overlapping chunks of text.
Step 3 - Embed each chunk into a dense vector using an embedding model.
Step 4 - Store all vectors in a vector database like FAISS or Pinecone.
Step 5 - At query time, embed the user query using the same embedding model.
Step 6 - Search the vector database for the top-K most similar chunks.
Step 7 - Pass the retrieved chunks as context to the LLM.
Step 8 - The LLM generates an answer grounded in the retrieved context.

FAISS (Facebook AI Similarity Search) is an open-source library developed by
Meta Research for efficient similarity search and clustering of dense vectors.
FAISS can search through millions of vectors in milliseconds. It supports
multiple index types including IndexFlatL2 (exact brute-force search using
L2 distance), IndexIVFFlat (inverted file index for approximate search),
and IndexHNSWFlat (hierarchical navigable small world graphs).
FAISS is the most widely used vector search library for RAG systems.

Vector Embeddings are numerical representations of text as fixed-size arrays
of floating point numbers. Semantically similar texts produce similar vectors.
The cosine similarity or L2 distance between two vectors measures how related
they are. Embeddings are generated by transformer-based models like
SentenceTransformers, OpenAI text-embedding-ada-002, or BGE models.

SentenceTransformers is a Python library built on HuggingFace Transformers
for computing dense vector embeddings of sentences and paragraphs. It uses
pre-trained models fine-tuned on semantic similarity tasks. The model
all-MiniLM-L6-v2 produces 384-dimensional embeddings and is fast and lightweight.
Larger models like all-mpnet-base-v2 produce 768-dimensional embeddings with
higher accuracy but slower inference speed.

Chunking is the process of splitting large documents into smaller pieces before
embedding. Chunk size controls how many characters each chunk contains.
Chunk overlap ensures continuity between consecutive chunks so no information
is lost at boundaries. Typical chunk sizes range from 256 to 1024 characters
with 50 to 100 character overlap.

Prompt Engineering is the practice of designing effective input prompts for LLMs.
A well-structured prompt includes a system instruction, retrieved context,
and the user question. Grounding the prompt in retrieved facts significantly
improves answer accuracy and reduces hallucination.

Hallucination in LLMs refers to the model generating confident-sounding but
factually incorrect information. RAG mitigates hallucination by constraining
the LLM to only use information present in the retrieved documents.
A retrieval gate can further filter queries where no sufficiently similar
document chunk is found, refusing to answer rather than hallucinating.
